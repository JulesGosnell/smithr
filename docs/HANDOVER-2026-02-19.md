# Skald Handover — 2026-02-19

## What Happened Tonight

The smithr-sonoma.img disk image was swapped from artha-sonoma.img earlier, which caused an iOS simulator runtime mismatch regression. Xcode 16.2 SDK (build 22C146 = iOS 18.2) couldn't find a compatible runtime because Apple only offers iOS 18.3.1 (build 22D8075) for download.

### What Was Fixed
1. **Runtime downloaded**: iOS 18.3.1 (22D8075) — 8.1GB
2. **Runtime match override applied**: `xcrun simctl runtime match set 'iphoneos18.2' '22D8075'`
3. **RuntimeMap.plist installed** at both user (`/Users/smithr/Library/Developer/CoreSimulator/`) and system (`/Library/Developer/CoreSimulator/`) levels
4. **All stale state cleaned**: artha-ios user deleted, RAM disk LaunchDaemon removed, /tmp cleaned
5. **Persisted via clean shutdown** in persistent mode, verified on volatile reboot
6. **iOS simulator build succeeded** — Artha confirmed green

### What Was Created
- `artha-ios-simulator` and `artha-ios-device` macOS users on the VM (volatile — will need recreation on restart)
- These were created manually via SSH, NOT through the Smithr lease API

### What's Broken / Incomplete
- **`create-build-user.sh` has a bug**: The `sudo` copy steps for SSH keys and RuntimeMap fail silently. I had to run the setup commands manually after user creation. The script needs debugging — likely a PATH or permission issue when invoked via SSH.
- **Workspace creation for iOS not tested via API**: Artha's iOS workspaces were created manually. The correct flow is: `POST /api/leases` with `workspace=artha-ios-simulator&platform=macos&lease_type=build`. This should auto-create the macOS user via `smithr.macos/ensure-user!`.
